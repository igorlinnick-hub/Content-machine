01

Current Problems

The existing UI has three compounding issues that make it feel like a scaffolded prototype rather than a shipped product.

🌫️
Critical

GradientBg fails silently

The current implementation uses position:fixed; filter:blur() on child divs. GPU compositing layers in Safari and Chrome can promote these elements off the main stacking context, causing the blobs to render behind the viewport clipping layer — the gradient simply doesn't appear. No errors, no fallback.

📋
High

Flat list layout feels generic

Posts, scripts, and arsenal items are rendered as uniform tall rows. No hierarchy, no information density variation. The user can't scan a page at a glance — every row demands equal attention.

🔲
Medium

Buttons lack tactile presence

Action buttons use flat bg-sky-500 fills. There's no frosted-glass treatment, no depth from inset highlight, no sense of surface material. On a gradient background they'll appear as opaque stamps rather than surfaces.

🎨
Cascade

No consistent motion language

Animations exist (cm-rise, cm-fade-in, shimmer, ping) but they don't form a system. The background gradient, card entrance, and button feedback all use different easing curves and durations with no shared vocabulary. Once the gradient is fixed, the motion needs to be audited against a single timing token set.

03

Implementation Tasks

Ordered by dependency — gradient fix first (unblocks everything else that sits on top of it).

  • Fix animated gradient — move to body CSS
    Replace GradientBg's filter:blur blob approach with a body-level background-size:400% + animation that shifts background-position. Guaranteed to render in all browsers because it doesn't create a new compositing layer. Remove conflicting html,body { background: var(--background) } override. Simplify GradientBg.tsx to return null.
    globals.css GradientBg.tsx
    P0
  • Liquid-glass button variant — .cm-glass-btn
    Add a new globals.css class. backdrop-filter: blur(16px), border: 1px solid rgba(255,255,255,0.18), inset top-edge highlight via box-shadow. Replace primary CTA buttons on /posts and /arsenal with this variant. Test readability on both light and dark sections of the gradient.
    globals.css PostCard.tsx ArsenalRow.tsx
    P1
  • Bento-grid layout for /arsenal page
    Replace the uniform list with a 2-column CSS grid (gap: 12px). Cards: ~200px tall, dark surface (#111115), icon top-left, title + 1-line desc, status badge bottom-right. Wide cell for featured/pinned scripts. Hashtag chips below description. Entrance animation: cm-fade-in staggered by index.
    arsenal page CSS grid new component
    P2
  • Status badge component — reusable across bento + list
    StatusBadge.tsx: props = variant ('live'|'active'|'beta'|'updated'|'draft'). Each variant maps to a color token (teal=live, sky=active, indigo=beta, amber=updated, zinc=draft). Pill shape, 10px font, uppercase, dot + label. Used in BentoCard and ArsenalRow.
    new component design token
    P2
  • Motion audit — unify timing tokens
    Define three CSS custom properties: --ease-out: cubic-bezier(0.16, 1, 0.3, 1), --dur-fast: 120ms, --dur-medium: 280ms, --dur-slow: 550ms. Update all existing keyframe animations to reference these tokens. Ensures the gradient drift, card entrance, and button press all feel like one system.
    globals.css :root tokens
    P3
  • prefers-reduced-motion coverage for new animations
    Add body-gradient and any new keyframes to the existing @media (prefers-reduced-motion: reduce) block in globals.css. Freeze gradient at 50% 50%, disable bento stagger, set button transitions to 0ms.
    accessibility globals.css
    P3
